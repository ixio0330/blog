XSS란, 악성 스크립트를 사용자가 실행하도록 하는 공격 방법이다.

## XSS의 종류

1. 반사형 XSS: 악성 스크립트가 포함된 URL을 사용자가 클릭하고, 공격 대상 서버가 이를 그대로 반사해 응답 페이지에 포함시키는 경우 사용자 브라우저에서 악성 스크립트가 실행되는 공격 유형이다.
2. 저장형 XSS: 악성 코드를 공격 대상 서비스 DB에 저장하고, 사용자가 조회했을 때 스크립트가 실행되는 공격 유형이다.
3. DOM 기반 XSS: 악성 스크립트가 서버에 전송되지 않으며, 보안에 취약한 JS 코드로 인해 악성 스크립트가 실행되는 공격 유형이다.

## 대표적인 XSS 대응 방법

1. 이스케이프
2. 속성 값 따옴표로 감싸기 + 이스케이프
3. uri http/https 스키마 검증
4. 쿠키 httpOnly 처리 > XSS 공격으로 쿠키 획득을 차단
5. 적절한 DOM API 사용
6. Sanitizer API 사용 > XSS을 발생시킬 수 있는 문자열 제거

## CSP를 사용한 XSS 대응

CSP는 서버에서 허용하지 않은 스크립트, 리소스 불러오기 등을 제한하는 브라우저 기능이다.

요청 헤더, meta 태그 등을 통해 설정할 수 있으며 옵션 값은 `directive`라고 한다. 스크립트, 이미지 등의 각각의 리소스를 불러오는 것에 대한 설정을 할 수 있다.

### CSP가 활성화되면?

1. same origin도 차단되므로 내 서버에 있는 리소스를 가져오려면 `self`를 명시해야 한다. 만약 서브 도메인의 리소스를 가져와야하는 경우라면 `*.example.com` 와 같이 명시해준다.
2. 인라인 스크립트 실행이 차단된다. 활성화 하고 싶다면 unsafe-inline 옵션을 사용하면 되지만 권장하지 않는다.

### Strict CSP

안전하게 인라인 스크립트를 실행하는 방법 2가지와 함께 Strict CSP를 소개한다.

1. nonce-source: 랜덤 토큰을 발행해 헤더에 있는 토큰과 script의 nonce 속성값이 토큰이 일치하는 경우에만 인라인 스크립트를 실행한다.
2. hash-source: 인라인 스크립트 자체를 해싱하여 전송하고, 해시값을 비교해 무결성이 검증된 경우에만 인라인 스크립트를 실행한다. 정적인 웹사이트에서 사용할 수 있다.
3. strict-dynamic: 동적으로 스크립트 태그를 생성하고 실행할 수 있는 옵션이다.
4. base-uri, object-src: base 태그를 차단하고, flash와 같은 외부 프로그램 실행을 차단한다.
