---
title: 방방곡곡 앱스토어 출시하다!
tags: [개발]
---

작년에 개발 챌린지를 하면서 만든 <a href="https://bbgg.dasoslab.com/" target="_blank">방방곡곡</a>이라는 서비스가 있다. 공공데이터 OpenAPI를 사용해 축제 정보를 지도 기반으로 보여주는 서비스다. 한달 안에 앱을 만들어서 출시하는걸 목표로 한 챌린지여서 빠르게 만들어서 플레이 스토어에 출시 한 후 한동안 방치하고 있었다. 오랜만에 생각이 나서 firebase에 들어가보니 적지만 유저가 존재했고, 축제 이미지가 제대로 나오지 않는 버그가 있어서 빠르게 고쳐서 플레이 스토어에 배포했다. 이번에 배포를 하면서 꾸준히 앱을 업데이트하고 운영해야겠다 싶었고, 그동안 엉망인 앱을 사용했을 유저들에게 미안했다.

버그를 고치는 김에 기능도 이것저것 추가했고, 원래는 별도의 서버 없이 앱에서 공공데이터 OpenAPI를 바로 조회했는데 본격적으로 서비스를 운영을 하기 위해 프로덕션 목적으로 OpenAPI 신청을 다시 하고 새로 API 키를 발급 받아서 proxy를 뒀다. proxy는 vercel에 배포했는데 vercel CI/CD는 github 조직 계정의 경우 유료 플랜 유저만 사용할 수 있어서,, 나는 토큰을 생성해서 github actions로 proxy 폴더에 변경이 발생했을 때만 배포하도록 트리거를 설정해뒀다.

그리고 애드몹 광고도 붙이고 스토어 스크린샷도 새로 만들어서 적용했다. 광고를 붙였으니 마케팅을 해서 유저수를 늘려보면 좋겠다 싶었는데, 마케팅을 하려면 앱스토어에도 배포하는게 낫겟다는 생각이 들었다. 그래서 앱스토어 개발자 계정을 등록하고, 랜딩 사이트 다듬어서 바로 심사를 넣었다.

앱스토어 심사는 2번 만에 통과했는데, 위치 정보 권한 요청 문구가 부실하다는 이유였다. 문구를 수정해서 다시 심사를 넣었더니 바로 승인됐다! 다양한 서비스들로 앱스토어 심사를 받아보면서 드는 생각인데 앱스토어 심사는 가챠다. 운이 좋으면 한번에 통과되고 운이 안좋으면(혹은 특정 카테고리는) 무한 리젝을 당한다. 그에 비해 플레이 스토어는 관대한 편이다.

앱스토어까지 출시를 했으니 본격적으로 마케팅을 해야되는데 리텐션 유지가 고민이 된다. 신규 유입이 늘어난다고 해도 현재는 축제 정보만 제공하다보니 리텐션이 떨어질 수 밖에 없어서 리텐션을 올릴 수 있는 기능을 추가해야 할 것 같다. 아자아자~!
